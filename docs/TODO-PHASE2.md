# Noodla Phase 2 — 実装予定一覧

> Phase 1 完了時点（2026-04-14）の未実装事項。
> Phase 2 の優先順位と実装方針を整理したドキュメント。

---

## スコープ外（Phase 2 には含めない）

以下は意図的に Phase 2 から除外する。

| 項目 | 除外理由 |
|------|---------|
| ❌ Stripe 決済統合 | アプリストア課金（IAP）前提のため後回し。In-App Purchase の設計が先 |
| ❌ AI API 接続（LLM直呼び） | バックエンド基盤整理が先。API Gatewayなしの直呼び構成は避ける |
| ❌ 実分散通信 | バックエンド基盤・プロトコル設計が先 |

## Phase 2 優先実装項目

### 🔐 1. 認証基盤

| タスク | 詳細 |
|--------|------|
| JWT認証 | アクセストークン + リフレッシュトークン方式 |
| OAuth対応 | Google / Apple Sign-In（AppStore 審査要件） |
| セキュアストレージ | `expo-secure-store` でトークン保存 |
| セッション管理 | トークン有効期限・自動リフレッシュ |
| ログアウト処理 | トークン破棄・状態リセット |

### 📱 2. 端末登録

| タスク | 詳細 |
|--------|------|
| デバイスID生成 | `expo-device` + UUID で端末識別子を生成 |
| 端末登録API | バックエンドに端末情報（OS、モデル名、UUID）を登録 |
| 端末認証 | デバイスIDとJWTを紐付けた二重認証 |
| 複数端末対応 | 同アカウントで複数端末の管理 |

### 🌐 3. ノード状態管理

| タスク | 詳細 |
|--------|------|
| バックグラウンド参加 | `expo-task-manager` + `expo-background-fetch` |
| Wi-Fi監視 | `@react-native-community/netinfo` でリアルタイム検知 |
| バッテリー監視 | `expo-battery` で充電状態・残量監視 |
| ノード登録・切断 | バックエンドとのWebSocket接続でリアルタイム状態同期 |
| 参加制御ロジック | Wi-Fi/充電/省電力条件に応じた自動ON/OFF |
| ハートビート | 定期的なpingでノード生存確認 |

### 💰 4. ポイント台帳

| タスク | 詳細 |
|--------|------|
| ポイントAPI | 残高取得・履歴一覧・加算/消費エンドポイント |
| リアルタイム同期 | WebSocket or Server-Sent Events でポイント変動通知 |
| 取引ID管理 | 冪等性保証のためのトランザクションID |
| ポイント有効期限 | 失効ルール設計と通知 |
| 消費バリデーション | 残高不足時のエラーハンドリング |

### 🏆 5. ランク台帳

| タスク | 詳細 |
|--------|------|
| ランクAPIとの接続 | スコア取得・ランク判定エンドポイント |
| ランクアップ処理 | 条件達成時のボーナスポイント付与 |
| スコア計算ロジック | 参加時間・タスク成功数・安定性の重み付け |
| グローバルランキング | 全ユーザーのリーダーボードAPI |

### 🏗️ 6. API骨格設計

| タスク | 詳細 |
|--------|------|
| REST API設計 | OpenAPI (Swagger) でエンドポイント定義 |
| 認証ミドルウェア | JWT検証・Rate Limiting |
| ユーザー管理 | CRUD + プロフィール更新 |
| エラーコード統一 | API エラーレスポンスの標準化 |
| ステージング環境 | dev / staging / prod の環境分離 |
| API クライアント | `src/api/` ディレクトリにAxios/fetch wrapper実装 |
